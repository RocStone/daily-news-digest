>

>

#### 5. NousResearch/hermes-agent（用户关注项目）
- **原文链接**: [GitHub链接](https://github.com/NousResearch/hermes-agent)

**一句话说清楚它是啥**: Nous Research做的一个开源AI助手（MIT协议，免费），跟Claude Code类似都是终端里跑的AI，但它最大的不同是——它会"长记性"。用得越久，它越懂你，而且它会把学到的东西变成可复用的技能。

**先说它和Claude Code最本质的区别，不然容易搞混**:
- **Claude Code**: 每次开新对话就是一张白纸（除了CLAUDE.md），它不记得上次帮你做过什么。它专注写代码，背后只能用Claude模型。
- **hermes-agent**: 它记得你们的所有对话历史，能搜索过去的经验，知道你是谁、喜欢什么风格。它不只是写代码，是个通用助手。而且你可以随时切换背后的模型——Claude、GPT、Gemini、Llama、Kimi、MiniMax，200+模型一条命令切换。

**它的"长记性"到底是怎么实现的——三层记忆，各管各的**:

**第一层：聊天记录全文搜索（情景记忆）**
> 所有跟它的对话都存在本地SQLite数据库里，建了全文搜索索引。三周前你让它帮你解决过一个CUDA的OOM报错，今天你又遇到类似的错，它会自动搜到那次对话，跟你说"上次我们是通过设置 `PYTORCH_CUDA_ALLOC_CONF` 解决的，要不要用同样的方案？"——而不是从零开始帮你排查。

**第二层：记住你是什么样的人（语义记忆）**
> 你用了一段时间后，它会慢慢建立一个关于你的"画像"：你喜欢用什么编程语言、你的代码风格是驼峰还是下划线、你做的是CV还是NLP、你常用的工具链是什么。这个画像跨会话持续更新。比如你连续几次都让它"别用print调试，用logger"，它以后就自动用logger了，不用你每次提醒。

**第三层：自动学会新技能（程序记忆）——这是最有意思的**
> 举个具体例子：你第一次让它"帮我把这个FastAPI项目打包成Docker镜像，推到我的私有仓库"。它花了好几步：写Dockerfile、构建镜像、tag、push，中间还帮你处理了认证问题。做完之后，它会自动把这整套流程抽象成一个"技能文件"存起来。下次你说"打包部署一下"，它不需要重新推理怎么做，直接调用上次学会的技能，几秒搞定。而且如果中间哪步出了问题，它还会自动优化这个技能。

**安装和日常使用长什么样**:
```bash
# 一行命令安装（Linux/Mac/WSL2）
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

# 选你要用的模型（不绑定任何一家）
hermes model

# 开始对话（打开一个终端UI，支持多行输入、Tab补全、流式输出）
hermes
```

**几个比较酷的额外功能**:
- **多平台消息网关**: `hermes gateway` 启动后，你可以同时在Telegram、Discord、Slack、WhatsApp上跟同一个Agent对话，所有对话记录互通。你在Telegram上跟它聊的内容，切到Discord它也记得。
- **定时任务**: 内置cron调度器。你可以设置"每天早上8点帮我扫一遍GitHub trending，把跟AI相关的项目总结发到我的Telegram"，它自动执行。
- **子Agent并行**: 可以同时开几个子Agent做不同的事，互不干扰。
- **远程执行**: 不一定跑在你本机。可以跑在Docker容器、SSH远程服务器、甚至Serverless环境（闲置时不花钱）。
- **40+内置工具**: 文件操作、代码执行、网页搜索、截图等等，开箱即用。

**对你来说值不值得试**: 如果你觉得Claude Code"每次新对话都不记得上次做过什么"很烦，或者你想要一个不绑定Claude的通用AI助手，可以试试。它的记忆系统确实比Claude Code现有的CLAUDE.md机制高级很多。缺点是它不是大公司的产品，稳定性和模型调用的兼容性可能不如Claude Code。

> 新的的问题：所以这是一个新的agent ai，无法和我现在订阅的claude code做结合对吗？
>
> 不完全对，分两个层面说：
>
> **第一，它能不能用 Claude 的模型？能。但用的不是你的 Claude Max 订阅额度。**
>
> hermes-agent 是"自带钥匙"模式（Bring Your Own API Key）——你自己去 Anthropic 官网申请一个 API Key，按 token 用量付费，跟你每月交的 Claude Max 订阅费是两套完全独立的计费系统。你的 Max 订阅只能在 claude.ai 网页版和 Claude Code CLI 里用，hermes-agent 吃不到这个额度。
>
> 所以如果你想在 hermes-agent 里用 Claude Opus，你要额外花 API 调用的钱。或者你也可以通过 OpenRouter 来调用，OpenRouter 聚合了 200+ 模型（包括 Claude、GPT、Gemini、Llama 等），一个 API Key 全搞定，按量计费。
>
> **第二，它和 Claude Code 能不能"结合"？不能，它们是两个完全独立的工具。**
>
> 打个比方：Claude Code 是 Anthropic 官方出品的一把瑞士军刀，专门为写代码优化，深度绑定 Claude 模型，好处是简单省心，坏处是不灵活（只能用 Claude、没有跨会话记忆、不能切模型）。
>
> hermes-agent 是 Nous Research（一个开源社区）做的另一把刀，它不跟任何模型绑定，主打的是记忆系统和自我进化能力。它们之间没有任何集成接口，不存在"让 hermes-agent 当 Claude Code 的插件"这种用法。
>
> **所以实际的选择是**：要么用 Claude Code（你现在的工作流，Max 订阅覆盖），要么用 hermes-agent（需要单独付 API 费用），或者两个都装着、不同场景用不同的。对你来说，除非你对"跨会话记忆"和"自动学技能"这两个特性有很强的需求，否则没必要折腾——Claude Code 的 CLAUDE.md + memory 机制虽然原始一些，但配合你现在的 vault 工作流已经够用了。

#### 6. dzhng/deep-research（用户关注项目）
- **原文链接**: [GitHub链接](https://github.com/dzhng/deep-research)

**一句话说清楚它是啥**: 开源免费（MIT协议），核心代码不到500行TypeScript。它就是OpenAI和Gemini那个"Deep Research"功能的开源平替——你给它一个研究课题，它自己去网上一轮一轮地搜、一层一层地挖，最后给你一份带引用链接的完整研究报告。

**先说清楚它和你平时做研究有什么不同——核心区别就一个字：递归**:

你现在用Claude Code的research skill或者WebSearch做研究，流程是这样的：搜一次 → 看结果 → 总结 → 完事。这是"单轮搜索"。

deep-research的做法完全不一样。打个最直白的比方：
> 你是一个写论文的研究生。你的导师（deep-research）让你去调研"2026年AI Agent的发展趋势"。
> - **第一轮（表层）**: 你搜了5个方向，每个方向找了几篇文章，读完之后你发现："哦，原来现在Agent的记忆机制是个热点，但我还不清楚具体有哪些技术路线。"
> - **第二轮（深挖）**: 你基于第一轮发现的新问题，又搜了5个更精准的方向，比如"Agent记忆架构对比""RAG vs 长上下文窗口在Agent中的应用"。这次搜到的东西比第一轮精准得多，因为你已经知道该问什么问题了。
> - **最终**: 把两轮的发现合在一起，写出报告。
>
> deep-research就是自动化了这整个过程。每一轮搜完，它不是傻乎乎地总结完就结束，而是会问自己："我还有什么不知道的？还有什么值得深挖的？"然后自动发起下一轮更精准的搜索。

**走一遍具体流程**:

1. **你输入**: "调研一下2026年开源AI Agent框架的竞争格局"
2. **你设两个参数**:
   - 广度 = 5（每轮同时搜5个不同方向）
   - 深度 = 2（最多递归挖2层）
3. **它先反问你几个问题**: "你更关注哪些方面？技术架构还是社区活跃度？""你关心的是编程类Agent还是通用Agent？"你回答后它才开始干活——这是为了避免搜出一堆你不想要的东西。
4. **第一层搜索**: 它生成5个搜索查询（比如"top open source AI agent frameworks 2026""hermes agent vs autogen comparison"等），同时用Firecrawl API去搜网页、抓内容。搜完后AI读完所有结果，提取关键发现，同时标注"这几个点还不清楚，需要继续挖"。
5. **第二层搜索**: 基于上一步标注的"不清楚的点"，再生成5个更精准的搜索查询，再搜一轮。这次因为带着上一轮的全部知识作为上下文，搜索质量会高很多——不会重复搜已经知道的东西。
6. **输出报告**: 把两层的所有发现编成一份结构化的Markdown报告，每条结论都带着来源链接。

**这一趟下来搜了多少次**: 5广度 x 2层 + 初始5次 = 大约15-25次网页搜索。比你手动搜效率高得多，而且搜索方向是AI自己根据前一轮的发现决定的，比你自己想关键词更全面。

**安装和用起来难不难**:
```bash
git clone https://github.com/dzhng/deep-research
cd deep-research
npm install
# 在 .env.local 里填两个key：
# FIRECRAWL_KEY=xxx （用来搜网页和抓内容）
```

#### 6. CodeYam CLI & Memory
- **原文链接**: [Product Hunt](https://www.producthunt.com/products/codeyam-cli-memory)
1. **核心洞察 (Core Insight)**: 为Claude Code等AI编程助手提供综合记忆管理，分析错误模式并生成精准的上下文规则。
2. **深度剖析与底层逻辑 (Deep Analysis)**: 现阶段AI编程最大的痛点是“金鱼记忆”和“死循环犯错”。CodeYam抓住了RAG（检索增强生成）在代码库场景下的痛点：不缺搜索能力，缺的是对“业务背景和历史踩坑经验”的全局记忆。它充当了一个伴生特工（Background Agent），从错误流中提取Meta-knowledge并固化到系统Prompt中。
3. **直觉建立 (Intuition Building)**: 就像给容易犯健忘症的天才大脑（Claude）外挂了一个错题本，并且每次他做题前都强行让他读一遍。
4. **批判性反思与盲区 (Critical Reflection)**: 随着原生模型Context Window（上下文窗口）越来越大，并且内置了类O1的长链自我反思机制，这种外挂式记忆工具的生存空间会越来越窄。
5. **一句话本质 (The Essence)**: 说穿了，这东西就是个自动帮你写和更新 `.claude.md` 系统提示词文件的脚本。

> 这是skill嘛？用起来实际的效果会怎么样？对记忆管理我非常感兴趣，要介绍的详细点，要有例子。
>
> **一句话说清楚它是啥**: 不是skill，是一个免费的命令行工具（`npm install -g @codeyam/codeyam-cli`），跑在你本地，不上传数据，不用注册。
>
> **先说它解决什么痛点——你肯定遇到过**: 你用Claude Code写代码的时候，有没有发现Claude总是犯同一个错？比如你的项目里数据库连接用的是自定义的wrapper，但Claude每次都用标准的SQLAlchemy写法，你纠正一次它好了，下次新对话又犯。你写了CLAUDE.md告诉它"用我们的wrapper"，但过了两周代码改了，CLAUDE.md里的说明已经过时了，你又懒得更新。这就是CodeYam要解决的问题：**自动帮你维护Claude的"记忆"，让它不再重复犯错**。
>
> **它具体怎么工作——走一遍完整流程**:
>
> **场景**: 你有一个Python项目，里面的测试用的是pytest + 你自己写的fixture体系，不是标准写法。
>
> **第1天**: 你跟Claude Code说"帮我给这个函数写测试"。Claude写了一个标准的pytest测试，没用你的自定义fixture，你纠正了它。对话结束。
>
> **后台自动发生的事（第一层：对话反思）**: CodeYam的后台Agent自动扫描了这次对话的transcript，发现了"Claude在测试文件上犯了错，用户纠正了它"这个模式。它自动创建了一条Rule：
> - 匹配路径：`/tests/**/*.py`
> - 规则内容："这个项目的测试必须使用自定义的 `conftest.py` 中的 `db_session` 和 `mock_user` fixture，不要用标准的 pytest 写法"
>
> **第2天**: 你开一个新对话，又让Claude写测试。Claude打开测试文件的时候，CodeYam自动把上面那条Rule注入到Claude的上下文里。Claude这次直接用对了你的自定义fixture，不用你再纠正。
>
> **第15天（第二层：规则审计）**: 你重构了 `conftest.py`，把 `db_session` 改名成了 `database`。CodeYam的审计Agent检测到 `conftest.py` 变了，自动去检查关联的Rule，发现Rule里还写着 `db_session`，于是自动更新Rule里的内容。如果不确定怎么改，就会标记为"需要审核"，等你在Dashboard上确认。
>
> **Dashboard（第三层）**: 你跑 `codeyam` 命令，浏览器弹出一个本地网页，上面列着所有的Rule——哪条Rule匹配哪些文件、最近改过没有、有没有需要你审批的。你可以手动编辑、删除、添加。
>
> **和你现在的CLAUDE.md到底有什么区别**:

|  | 你的CLAUDE.md | CodeYam Rules |
|---|---|---|
| 谁来写 | 你手动写 | 后台Agent自动从对话中提取 |
| 作用范围 | 全局的，Claude每次都读全部内容 | 按文件路径精准匹配，只在碰到相关文件时注入 |
| 保鲜度 | 你不更新就烂掉，而且你大概率懒得更新 | 代码改了它自动跟着更新 |
| 你需要做什么 | 想到什么写什么 | 什么都不用做，它自己学 |
>
> **值不值得用**: 如果你用Claude Code的频率很高、项目有很多"只有你知道的潜规则"（比如代码风格、特殊框架用法、不能碰的文件），那它能帮你省大量重复纠正的时间。它本质上就是一个"Claude的自动化错题本"——Claude犯过的错自动记录，下次自动提醒它别再犯，而且错题本还会跟着代码变化自动更新。

> 新问题：这个似乎看起来能和claude code结合？但是我claude犯错可不只是代码，还可能是科研的逻辑问题、输出的方式问题等等，这些都能自动记忆吗？它怎么判断什么东西要记忆？
>
> **先回答能不能和Claude Code结合**: 对，它就是专门给Claude Code做的，不是给别的AI用的。装好之后它会自动读取你和Claude Code的每次对话记录（transcript），你不需要做任何额外操作。
>
> **然后说你最关心的问题：非代码类的错误能记吗？**
>
> 老实说，**大概率不太行，至少现阶段不行**。原因如下：
>
> 1. **它的规则是按文件路径匹配的**。比如规则可以设成"当Claude碰到 `/tests/**/*.py` 这些文件时，注入这条提醒"。这意味着它天生就是围绕"代码文件"设计的——你的科研逻辑错误、输出格式问题，没有一个明确的"文件路径"可以挂靠。比如"Claude总是在分析论文时过于轻信作者的结论"，这个规则该匹配哪个文件路径？没有合理的匹配目标。
>
> 2. **它的反思Agent扫描的是"coding session transcripts"**。官方博客和HN讨论里，从头到尾说的都是"编码会话""代码库中的困惑模式""架构决策"——没有任何地方提到它能识别科研推理错误或写作风格问题。它的后台Agent本质上是在找"Claude在操作某段代码时犯了什么错"，不是在找"Claude在思考问题时逻辑哪里不对"。
>
> 3. **它怎么判断什么该记**：它的后台Agent会扫描对话记录，找三类东西——(a)Claude表现出困惑的地方、(b)你纠正Claude的地方、(c)关于代码架构的重要决策。注意，这三类都是围绕"代码"的。它不会去判断"Claude的论文分析逻辑有没有漏洞"。
>
> **那你的需求怎么解决？**
>
> 你说的"科研逻辑问题、输出方式问题"，其实你现在用的CLAUDE.md已经在做这件事了——比如里面写了"禁用双引号""帮助分析论文时要批判性思考"。CodeYam解决的是"CLAUDE.md懒得手动维护"的问题，但它只能自动维护代码相关的规则。
>
> 对于非代码类的规则（科研逻辑、输出格式、分析风格），目前最靠谱的方案还是：**你发现Claude犯了什么错，手动写进CLAUDE.md**。比如你发现Claude总是在分析新闻时写太多废话，就在CLAUDE.md里加一条"新闻分析要简洁，每条不超过X行"。虽然原始，但对你这种非纯编码场景来说，反而是最精准的。
>
> **总结**: CodeYam对纯编码项目很好用，但你的vault是知识管理+科研+新闻分析，不是纯代码项目，它的自动记忆机制覆盖不到你的核心痛点。你的CLAUDE.md手动维护的方式，对你的场景反而更合适。

2. **深度剖析与底层逻辑 (Deep Analysis)**: 典型的“帝国过度扩张（Imperial Overstretch）”困境。美国既要在欧洲对付俄罗斯，又要在中东摁住伊朗，还要在亚太遏制中国，同时国内顶着通胀和天量利息。在多线作战的压力下，其军事威慑力和美元霸权正在遭受冷战后最严峻的实质性考验。
3. **直觉建立 (Intuition Building)**: 就像一个欠了一屁股高利贷的黑老大，发现三个不同街区的小弟同时造反，手里的棒子根本不够用，只能拆东墙补西墙。
4. **批判性反思与盲区 (Critical Reflection)**: 这种论调在国内舆论场常带有情绪化的幸灾乐祸。美国的底层科技创新力（如AI领域的断层领先）、全球金融吸血能力以及粮食能源的绝对自给自足，赋予了它极强抗击打能力。“撑不住”是一个极度漫长的衰退过程，而非瞬间的崩塌。
5. **一句话本质 (The Essence)**: 说穿了，这东西是霸权国家在多极化冲击下，捉襟见肘的战略疲态显现。
